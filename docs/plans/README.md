# docs/plans — PR briefs

Un fichero por PR, siguiendo `_TEMPLATE.md`.

**Convención:** `PR-NN-<slug-corto>.md` — ej. `PR-00-fix-collection-errors.md`,
`PR-A1-admin-creds-summary.md`.

**Workflow:** ver `docs/MVP-ROADMAP.md §2`.

**Regla:** el brief se escribe **antes** de tocar código. Aprobado por el
humano = lock de scope. Lo que no esté en el brief va a otro PR.
